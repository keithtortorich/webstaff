Reporting a Vulnerability

We take the security of your office infrastructure seriously. If you discover a potential vulnerability, please report it to the engineering team immediately using the following process:

How to Report

Please send an email to security@webstaff.enterprise with the subject line: Vulnerability Report: [Short Description].

Please include the following information in your report:

Description: A clear, concise description of the security concern.

Steps to Reproduce: A step-by-step guide on how to trigger the vulnerability.

Impact: A description of the potential risk or unauthorized access possible.

Environment: Details about your specific cloud configuration (e.g., Cloud Run, local Docker, Firestore region).

What to Expect

Acknowledgment: You will receive a response within 48 hours acknowledging receipt of your report.

Triage: The team will analyze the report and provide an estimated timeline for remediation if confirmed.

Transparency: Once a fix is verified, we will coordinate with you to ensure the patch is safely deployed. If the vulnerability is declined (e.g., intended behavior or non-security issue), we will provide a detailed explanation of our reasoning.

Thank you for helping us keep WebStaff_4 secure for all home service contractors.
